Scan converting triangles

Scan convert the the four triangles ABC, ADB, BEC, ACF, and ADG shown below and in slide33 of Lecture 3, into a 400 by 400 pixel BMP image. This means color in the pixels inside or on the (appropriate) edges of each tringle by the same opaque or transparent color. For the triangles shown below, and a blue transparent color on a black background, the result should look like the image at the bottom of this assignment.


A = (74.5, 175.5), B = (224.5, 325.5), C = (374.5, 175.5), D = (74.5, 325.5), E = (374.5. 325.5), F = (150, 100), and G = (25, 125). Make all the triangles the same bright color of your choice, over a black background. Make two BMP images, one using set_pixel(x, y, r, g, b) or set_pixel_alpha(x, y, r, g, b, 1.)  for opaque triangles, and one using set_pixel_alpha(x, y, r, g, b, a) with a = .5, to verify that no pixels are drawn twice. You can do the verification by using the snipping tool in Windows or the Screenshot tool in Linux to copy a displayed version of your BMP file and enlarge it to search for bright pixels. (I will probably do that when I grade the homework.) Please upload your .cpp source files. There can be two, or just one with an if test to determine whether to to call set_pixel(x, y, r, g, b) or set_pixel_alpha(x, y, r, g, b, a). If you upload only one .cpp file, also upload a README file telling be how to change between the options. Also upload the two .bmp output files.
