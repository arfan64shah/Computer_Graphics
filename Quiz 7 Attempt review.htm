Question text

The in-class exercise in Week 7 on Moodle, called "Changing the orthogonal window size using arrow keys" describes an interactive method to set the size variable in projMatrix.setOrtho(-size, size, -size, size, 0, 2) by using the left and right arrow keys. The initial view produced by the program, before using any of the arrow keys looks like this.


After I pressed the left arrow key a lot of times, the triangles got larger, and the blue triangle filled the screen. When size passed 0, and became negative, the blue shading suddenly changed slightly, and then the triangles got smaller again, looking like the image below.


This looks like a slightly smaller version of the original image above it, but rotated 180 degrees. Explain why this rotation occurs. 

If I understand Moodle Quiz settings correctly, you may either type your answer on this Moodle page or upload a file of any filetype with your answer. Please do not use a .rar file though, since I cannot read them. Good file types are .doc, .docx, .ppt, .pptx, .txt, .odt, or any image file that your camera produces of a handwritten answer.

Feedback

The formulas for the entries on the diagonal of this matrix, given in cuon-mult.js

in examples/lib, or in slides 27 through 30 in Lecture 16, in Moodle, Week 7, say that

the scale factor for x is 2/(right - left) and the scale for y is 2/(top - bottom).

If top and right are both size, and left and bottom are both -size, these entries are

both 2/(size - (-size)) = 2/(2*size) = 1/size. When size becomes negative, these scales

become negative, so the x and y coordinates are both flipped in sign. This double sign

flip is equivalent to a rotation of 180 degrees.
